## Шаг 2: Как писать Bug Reports в GitHub

Bug reports оформляются как Issues в интерфейсе GitHub в репозитории kompotai/bug-reports. Вот пошаговая инструкция:

### 2.1 Найдите раздел Issues

1. Перейдите в репозиторий: https://github.com/kompotai/bug-reports
2. Нажмите на вкладку **Issues** в верхнем меню
3. Нажмите зелёную кнопку **New Issue**

### 2.2. Заполните Bug Report

Используйте форму https://github.com/kompotai/bug-reports/blob/main/.github/ISSUE_TEMPLATE/bug_report.yml


### 2.3. Пример Bug Report

Основная структура:

Заголовок (Title):
- Краткое описание проблемы (1 строка)
Описание (Description):
Краткое описание проблемы и почему это баг.

Наглядный пример:
**Заголовок:**
Не сохраняется номер телефона при создании контакта
**Описание:**
При создании нового контакта номер телефона не сохраняется в системе, хотя поле заполнено корректно.

## Советы по поиску багов

- **Проверяйте основные сценарии:** регистрация, логин, создание/редактирование/удаление данных
- **Тестируйте граничные случаи:** пустые поля, очень длинные строки, спецсимволы
- **Обращайте внимание на UI:** сломанная вёрстка, кривые кнопки, неправильные тексты
- **Проверяйте валидацию:** что происходит при вводе некорректных данных
- **Тестируйте разные браузеры:** Chrome, Firefox, Safari

## Карьерный рост в команде

В нашей команде существует три уровня развития:

1. **Bug Reporter** — поиск и репортинг багов (ваш текущий уровень)
2. **Test Fixer** — исправление и улучшение автоматических тестов
3. **Developer** — разработка новых функций и архитектурных решений

**Важно:** Переход на следующий уровень осуществляется только Project Manager'ом на основе вашей работы и прогресса. Не стесняйтесь демонстрировать свои результаты и задавать вопросы!
